# 项目需求文档：Oracle PL/SQL 智能格式化引擎 (v2.0.0)

## 1. 项目概述

本项目旨在构建一个专为 Oracle 存储过程及复杂 PL/SQL 代码设计的专业格式化工具。区别于通用的 SQL 格式化库，本项目采用**自研的关键字对（Keyword-Pair）状态机引擎**，旨在解决 PL/SQL 格式化中常见的缩进错乱、`CASE...WHEN` 逻辑对齐困难、`JOIN` 链垂直排版及注释/字符串保护等行业痛点。

## 2. 核心功能需求

* **智能状态机引擎**：
* 基于词法分析，支持对 `BEGIN...END`, `IF...END IF`, `CASE...END CASE`, `LOOP...END LOOP` 等块结构的深度缩进处理。
* 实现“智能对齐”：支持 `SELECT`, `FROM`, `JOIN` 链的垂直对齐，并自动优化 `WHERE` 子句中多条 `AND/OR` 条件的换行布局。


* **高阶排版规范**：
* **强制换行规则**：`JOIN` 系列关键字（`LEFT/INNER JOIN` 等）强制换行；`EXIT WHEN` 保持原子性，禁止拆分。
* **缩进规范**：全代码采用统一的 4 空格缩进，且支持多层级嵌套下的相对对齐。
* **代码规范化**：自动将 SQL 关键字转换为全大写（字符串和注释内容除外）。


* **健壮性与保护机制**：
* **占位符保护机制**：在格式化全过程中，对单行注释 `--`、多行注释 `/* */`、以及引号内的字符串内容进行原子保护，确保格式化后内容不丢失、不乱码。
* **特殊字符防误伤**：在还原复合关键字（如 `END IF`）时，确保不会误伤代码中原始存在的下划线（如表名 `WMS_PROD`）。



## 3. 技术栈建议

* **后端**：FastAPI（高性能异步框架），处理前端发起的格式化请求。
* **前端**：Web 页面集成 **Monaco Editor**（与 VS Code 同内核），提供高亮显示、行号支持及代码实时编辑功能。
* **核心算法**：完全自研的原生 Python 状态机逻辑（无依赖外部复杂 AST 解析库）。

## 5. 附录：核心开发规范 (v2.0.0 新增)

* **Keyword 隔离符规范**：所有复合关键字必须通过 `@@` 占位符进行预处理，禁止在还原阶段使用 `replace('_', ' ')` 这种无差别替换。
* **断行逻辑遵循“前置处理”原则**：在分词流处理阶段，必须优先处理关键字的原子性，确保状态机在读取 `tokens` 时，关键字与标点符号（如 `,`, `;`) 的依附关系逻辑清晰。
* **注释移动策略**：必须支持注释随代码块层级进行智能缩进（例如：多行注释内部的缩进应与代码首行保持一致）。
